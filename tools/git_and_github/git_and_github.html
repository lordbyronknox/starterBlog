Git

  • git clone -a [clone repo with all branches]
  • git clone -b [clone specific branch]
  • Git branch [create a new local branch]
  • git checkout -b branch-3 [create branch and switch to it]
  • git branch -d [delete a local branch]
  • git reset —soft HEAD~1 [delete most recent commit, keeping the work you’ve done]
  • git reset —hard HEAD~1 [delete most recent commit, along with current work you’ve done]
  • git pull origin main [pull from the remote main branch. Make sure you are in local main]
  • git merge --abort [abort a merge]
  • git rebase —abort [abort a rebase]
  • git push --no-verify [push, without running tests]
(Note: to get out of the editor after ‘git tag’, hit escape, then ‘q!’)


DEPLOYING

BEFORE MERGING

Tag the branch (not main)

  • - git tag v0.0.3-1
  • - git push origin tag v0.0.3-1 --no-verify

    • Wait for test to complete. When successful merge the branch with main


    AFTER MERGING

    Check out main

    Pull latest changes

  • Tag, e.g. v0.0.3
  • Push tag

    • Wait for prod deployment

    (Note: if deployment fails and new changes are pushed - you must re-tag the branch. (Eg. git tag v0.0.3-2) )

    MERGE CONFLICTS

    NB. To avoid merge conflicts!!!! Always make sure your local ‘main’ is up to date with ‘remote’ main. - go to local ‘main’ and pull from remote ‘main’. Then create a new branch locally, and work on that new branch only.

    If you have a merge conflict:

    - Got to local ‘main’ and pull

  • git pull origin main

    • - Switch to the local branch (that you want to merge main into)

  • git checkout branch-3

    • - Merge the branch with main.git merge main

    - Fix the conflicts: Open the files with conflicts and choose the changes to keep. ‘Current’ is the latest changes.‘Incoming’ is the change coming from the pull (I.e. the old code you usually don’t want)

    - Commit and push
